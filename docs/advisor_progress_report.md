# 导师汇报：CycleState 研究进展

## 1. 核心想法：信号灯路口轨迹预测是一个多尺度时间依赖问题

我目前的工作围绕一个核心判断展开：

> 信号灯路口的轨迹预测，本质上不是单纯的轨迹外推，而是一个**多尺度时间依赖建模**问题——未来轨迹受到三种不同时间尺度上状态演化的共同约束。

具体来说，在信号灯路口，决定一辆车未来轨迹的不仅是"它自己过去几秒怎么动、周围车怎么交互"，还有更长时间尺度上的结构性力量：

- **短时尺度（秒级）**：个体车辆的运动学与邻居交互——这是现有方法普遍覆盖的。
- **中时尺度（十秒到分钟级，跨信号周期）**：车道级的排队形成、传播与释放波。绿灯亮起后，释放波从停止线向后传播，队列逐渐消散——这个动态过程持续的时间远超单个预测窗口，但它的演化规律会直接影响预测窗口内每辆车的启停行为。
- **长时尺度（分钟级，多个信号周期）**：信号周期的相位节律。红灯、绿灯、黄灯的周期性切换，以及每个相位已持续的时间、距离切换还有多久，构成了一个宏观的时间结构，这个结构约束了前面两个尺度上状态演化的方向。

**现有方法的不足**：无论是 Social LSTM、LaneGCN、Trajectron++ 还是 Scene Transformer，这些方法主要覆盖的是短时尺度（微观运动与交互）。即便一些信号灯路口专门方法（如 D2-TPred）把交通灯信息纳入了输入，它们通常也把它当作一个静态条件特征，而没有显式建模中时和长时尺度上交通状态如何随时间演化。

**我的核心假设**：如果模型能够同时记住并预测这三种时间尺度上的状态演化——车道级排队怎么变、信号周期怎么推进——那么对个体轨迹的预测就会更准、更可解释。

## 2. 方法实现：micro-meso-macro 分层框架

基于上述想法，我在 D2-TPred 的微观建模能力之上，搭建了一个三层框架：

- **Micro 层**：保留 D2-TPred 原有的轨迹编码器与图交互模块，负责短时尺度的个体运动与邻居交互。
- **Meso 层**：新增 queue-state 分支，从同车道车辆中提取排队密度、等待比例、释放比例、队列长度等统计量，通过 queue LSTM 建立车道级排队状态记忆。
- **Macro 层**：新增 cycle-state 分支，将相位 one-hot、已持续时长、剩余时长、相位切换标志编码为信号周期状态记忆。

**最关键的机制——Phase-Rolling Queue Memory**：我不让 meso/macro 状态在编码后被"冻结"，而是在预测期内持续滚动更新。具体来说，预测期的每一步，queue-state 都会根据上一帧的 meso state、当前相位状态、以及预测运动偏移，继续向前推演。这相当于让模型不仅编码了"当前排队是什么样的"，还在预测"未来排队会怎么演化"。**这是整个方法里最有辨识度的设计**，也是"多尺度时间依赖"这个核心想法的直接落地。

除此之外，方法还有一些配套设计：

- **Lane-Consensus Anchor**：同车道车辆共享一个车道级共识参考，避免个体噪声污染中观状态演化。
- **Phase-Conditioned Gating**：让信号灯状态调制 meso/macro 状态对解码的作用方式，而不是简单拼接。
- **Baseline-Compatible Decoder Residual**：以残差方式注入状态记忆，保护 D2-TPred 预训练好的 decoder 能力不被新结构破坏。
- **分阶段训练协议**：warmup（先学会状态记忆）→ refine（降低辅助损失权重，让状态记忆服务于轨迹预测）→ adversarial（小权重 GAN 精修），配合结构化辅助损失对 queue/cycle 状态进行分项监督。

## 3. 当前实验进展

- **原型已完成**：CycleState 模型可以正常训练和推理，支持从 D2-TPred 预训练 checkpoint 进行 warm-start。
- **训练协议已跑通**：分阶段训练、辅助损失、消融开关均已接入并验证。
- **关键 bug 修复**：发现并修复了两处训练态与推理态行为不一致的问题（step-0 rollout 使用了 teacher-forced future offset；decoder 直接将 rollout hidden 替换为 queue context 而未锚定观测期状态）。修复后，20-batch 短协议下 rollout-on 已重新优于 no_rollout 对照（ADE 66.8 vs 71.9，FDE 132.2 vs 141.0），说明方法主线没有概念性失败。
- **协议硬化已完成**：训练内验证已改为 true `val` split，`lr` 参数真实生效，并加入了 `grad_clip`、bounded rollout residual injection、warmup rollout-state detach 和状态稳定性日志。
- **基础交互建模 P0 修复已完成**：进一步代码审查发现两个更基础的问题并已修复：
  - `seqGAT` 前向被 `torch.no_grad()` 包裹，局部时序图注意力实际上没有参与训练；
  - `relation_Matrix` 在正常方向区间下会把距离内邻居无条件连边，削弱方向扇区约束。
  对应梯度流和方向扇区测试已补齐；本轮又加入了 `teacher_forcing_ratio` 显式协议开关测试，当前协议测试共 `39` 项，全部通过。
- **最新 true-val 结果**：
  - Stage 24：`50b = 87.082 / 175.723`，`100b = 231.420 / 420.862`
  - Stage 25（修复后）：`50b = 88.956 / 175.380`，`100b = 204.730 / 375.054`
  这说明 P0 修复确实缓和了 `100b` 崩坏，但仍未通过“100b 相对 50b 恶化不超过 15%”的稳定性门槛。
- **Stage 26 最小变量结论**：
  - 单独把 warmup 的 `teacher_forcing_ratio` 从 `0.8` 降到 `0.6`，并不能解决问题：`50b = 92.735 / 182.898`，`100b = 266.671 / 468.079`，反而更差。
  - 但把训练协议切成 `50b warmup -> 50b refine` 后，quick true-val 提升到 `84.772 / 170.878`，完整 `val + num_samples=20` 复核达到 `75.078 / 154.690`。
- **当前瓶颈**：问题已经从“有没有基础实现 bug”进一步收敛为“warmup 是否承担了不该由它承担的长链状态耦合训练”。因此当前更可信的方向不是继续在 warmup 里压 teacher forcing，而是让 refine 更早接管状态记忆到轨迹解码的对齐。

## 4. 下一步计划

1. **先做正式可比复核**：对 `warmup50_refine50_p0_seqgat_relation_v1` 跑 `test + num_samples=20`，确认它在最终 split 上是否仍保持当前优势。
2. **围绕当前最佳候选做消融**：按 `rollout -> decoder residual -> lane anchor -> state gating` 的顺序，验证哪些机制真正支撑了 `75.078 / 154.690` 这条证据线。
3. **补齐 baseline 审计口径**：补全 baseline 在 `val/test + num_samples=20` 的结果，严格区分“仓库可比线”和“论文表格线”。
4. **最终目标**：在不引入额外通用结构的前提下，用 `full-cycle traffic-state memory` 的主线拿到一条稳定、可复现、可解释的正式 test 结果，再判断是否有机会进一步逼近或超过 baseline。
